# CHance Language Specification (H27)

This document describes the H27 language mode implemented by `chancec` in this repository.
H27 is the current/default language standard (`-H27`).

H26 compatibility mode is available with `-H26` and enforces a subset (notably: no `union`, `packed`, `ref`, `object`, `new/delete`, `managed/unmanaged`, `try/catch/finally`, `throw`, `jump`, `is`, `===`, and no `--implicit-void-function`).

## 1. Toolchain and Outputs

- Front-end compiler: `chancec`
- Source files: `.ce`
- Project files: `.ceproj`
- Bytecode module: `.ccb` (textual ChanceCode IR)
- Library package: `.cclib` (metadata and optional embedded `.ccbin`)
- Backend compiler: `chancecodec`
- ARM64 assembler path: `chs`
- Linker path: `cld` or host compiler path depending on target flow

`chancec` parses source, runs semantic checks, emits `.ccb`, and optionally drives backend assembly/object/executable generation.

## 2. Language Modes

- `-H27`: default mode, full currently-implemented language
- `-H26`: compatibility mode for the H26 surface

Mode can be set in CLI args or `.ceproj` `args`/`control_args`.

## 3. Lexical Structure

### 3.1 Whitespace and Comments

- Whitespace separates tokens
- Line comments: `// ...`
- Block comments: `/* ... */`

### 3.2 Identifiers

- Start: letter or `_`
- Continue: letter, digit, `_`

### 3.3 Literals

- Integer literals: decimal, plus prefixed bases (`0x`, `0b`, `0o`, `0d`)
- Integer suffixes: unsigned (`u`/`U`) and width (`l`/`L` repeatable)
- Float literals: decimal/exponent; `f`/`F` for `f32`, default `f64`
- Character literals: C-style escapes including hex/unicode/octal support
- String literals: C-style escapes, adjacent literal concatenation

- `fun`, `action`, `ret`, `if`, `else`, `switch`, `case`, `default`, `while`, `for`, `match`, `break`, `continue`

Types and storage:

- `i8`, `u8`, `i16`, `u16`, `i32`, `u32`, `i64`, `u64`
- `f32`, `f64`, `f128`, `float`, `double`
- `long`, `ulong`, `int`, `uint`, `short`, `ushort`, `byte`, `ubyte`
- `void`, `char`, `bool`, `string`, `stack`, `static`, `constant`, `var`
- `struct`, `struc`, `enum`, `alias`, `object`, `ref`, `union`, `packed`

Modules and visibility:

- `module`, `bring`, `hide`, `expose`

FFI and interop:

- `extend`, `from`

Type/meta expressions:

- `as`, `is`, `sizeof`, `typeof`, `alignof`, `offsetof`, `null`

Attributes/metadata keywords:

- `chancecode`, `literal`, `entrypoint`, `jumptarget`, `export`, `raw`
- `preserve`, `section`, `forceinline`, `inline`, `hint`, `nohint`, `overridemetadata`

Memory/runtime control:

- `new`, `delete`, `managed`, `unmanaged`, `noreturn`

Exceptions and control transfer:

- `try`, `catch`, `finally`, `throw`, `jump`, `where`

## 5. Type System

### 5.1 Primitive Types

- Signed/unsigned integers: 8/16/32/64-bit
- Floating point: `f32`, `f64`, `f128`
- `char`, `bool`, `void`
- Built-in alias `string` as `char*`
- `va_list` recognized as intrinsic type

### 5.2 Compound Types

- Pointers: `T*`
- Arrays: `T[N]` and unsized `T[]`
- Structs: named and forward declarations
- Unions: named, forward declarations, and inline union forms in type contexts
- Function pointer types: `fun* (...) -> Ret`
- Action types: `action (...) -> Ret` (pointer-like callable type)
- Reference types (H27): `ref T`, `ref? T`, `ref! T` (nullability modes)
- Object type (H27): `object` (internally modeled as object-tagged pointer-like type)
- Imported qualified types: `Module.Type`

### 5.3 Generic Types and Constraints

- Function templates with type parameters
- Template constraints: `integral`, `floating`, `numeric`, `pointer`
- Template default types
- Generic aliases in pointer-pattern form (`alias Name<T> = T*...`)

## 6. Declarations

### 6.1 Functions

- Standard form: `fun name(params) -> Ret { ... }`
- Supports `hide`/`expose`, module-aware backend naming, and attribute-driven metadata
- `noreturn` supported and validated
- Function bodies:
- Normal block body
- `[ChanceCode]` body (statement-oriented IR text capture)
- `[Literal]` body (verbatim-style block capture with non-blank enforcement)

### 6.2 Variables

- Local and global variable declarations
- `var` requires initializer (inferred type)
- `constant` and `static` support
- Aggregate/initializer-list assignment at declaration

### 6.3 Struct/Union/Enum/Alias

- `struct` and `union` declarations and forward declarations
- `packed struct` layout support (`packed` keyword/attribute)
- Struct field defaults supported for literal int/float/string/null forms
- Enums define constants and scoped `Enum=>Member` aliases
- Type aliases and constrained generic alias pattern

### 6.4 Modules and Imports

- `module A.B;`
- `bring A.B;` and `bring A.B as Alias;`
- `hide` and `expose` visibility controls
- Module-qualified type lookup and import resolution

## 7. Expressions and Operators

### 7.1 Primary/Postfix

- Literals, identifiers, grouped expressions
- Calls and indirect calls
- Member access: `.`, `->`, `=>`
- Indexing `[]`
- Postfix `++`/`--`
- Casts: C-style `(Type)expr` and `expr as Type`

### 7.2 Unary

- `+`, `-`, `!`, `~`, `*`, `&`, prefix `++/--`
- H27: `ref expr` address-of synonym
- H27: `new Type` and `new Type[count]`
- H27: managed array adapter form `(managed[]:.length = expr) operand`

### 7.3 Binary/Other Operators

- Arithmetic: `+ - * / %`
- Bitwise: `& | ^ << >>`
- Logical: `&& || !`
- Comparison: `< <= > >= == !=`
- H27: strict equality `===`
- H27: type test `expr is Type`
- Assignment and compound assignment variants
- Conditional: `?:`

### 7.4 Built-in Expressions

- `sizeof`, `alignof`, `offsetof`, `typeof`
- `null`

## 8. Statements and Control Flow

- Block `{ ... }`
- Variable declarations
- `if`/`else`
- `while`
- `for`
- `switch`/`case`/`default`
- `match` expressions and match-driven statement lowering
- `break`, `continue`, `ret`

H27 additions:

- `managed { ... }` and `unmanaged { ... }` statement scopes
- `try { ... } catch (...) [where ...] ... finally { ... }`
- `throw expr;` and `throw expr -> payload;`
- `jump target_call(...);`
- `delete expr;`

## 9. Attributes and Metadata

Function/global metadata is parsed via bracket attributes (`[Name("...")]`) or keyword-attribute forms.

Implemented names include:

- `EntryPoint`
- `JumpTarget`
- `Export`
- `Raw`
- `Preserve`
- `Section("...")`
- `Inline`
- `ForceInline`
- `ChanceCode`
- `Literal`
- `OverrideMetadata("...")`
- `Hint` / `NoHint`

Notes:

- `ChanceCode` and `Literal` cannot be combined on the same function
- Some attributes are validated for declaration kind and body form

## 10. Preprocessor

Implemented directive family includes:

- `#define`, `#undef`
- `#if`, `#elif`, `#else`, `#endif`
- `#ifdef`, `#ifndef`
- `#error`, `#warn`, `#note`
- `#static_assert`

Built-in macros include compiler/version/platform information and file/line counters (for example: `__CHANCE__`, `__CHANCE_VERSION__`, `__LINE__`, `__FILE__`, target/platform macros).

The preprocessor avoids expanding inside strings/chars/comments and maintains recursion guards.

## 11. FFI and External Declarations

- `extend from "C" Ret name(params);`
- `extend fun name(params) -> Ret;`
- ABI defaults to C where applicable
- Varargs supported with final-position enforcement

## 13. CLI Highlights

- `--version` prints compiler version metadata
- `-H26` / `-H27` language mode selection
- `--implicit-voidp`, `--implicit-sizeof`
- `--implicit-void-function` (H27 only)
- backend and target selection (`-x86`, `-arm64`, `-bslash`, `--target-os`)

- library mode (`--library`)

## 14. Runtime/Stdlib Integration

When not explicitly freestanding, compiler driver wiring includes standard library/runtime library resolution and integration (`stdlib.cclib`, `runtime.cclib`) as part of the normal build flow.

## 15. H26 vs H27 Quick Delta

H27 introduces or enables in this compiler:

- `union` declarations/types
- `packed` struct declarations
- `ref` types and `ref` unary address-of form
- `object` type
- `new` and `delete`
- managed/unmanaged declaration and statement scopes
- exception features (`try/catch/finally`, `throw`, catch `where`)
- `jump` statement
- `is` operator
- strict equality `===`
- `--implicit-void-function`

Use `-H26` to disable these and remain within H26-compatible behavior.
